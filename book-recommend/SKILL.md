# Book Recommend — 英文阅读水平评估与图书推荐

根据微信读书英文书划线生词，自动评估英文阅读水平（CEFR），搜索并推荐匹配难度的英文原版书，
提供深度链接一键跳转添加到书架。

## 前置条件

- `weread-skills` 已安装
- `WEREAD_API_KEY` 环境变量已设置
- 微信读书中至少有一本英文书有划线记录

## 书架分组命名规范

微信读书书架中的英文书按 **Lexile + CEFR** 规则命名分组，实现难度分级管理。

### 命名格式

```
Lexile {CEFR级别}{难度偏移}
```

| 难度偏移 | 含义 | 示例 |
|----------|------|------|
| `-` | 该级别中偏易 | `Lexile B1-` |
| (无后缀) | 该级别标准难度 | `Lexile B1` |
| `+` | 该级别中偏难/过渡到下一级 | `Lexile B1+` |

### 完整分组列表（18 个）

| 级别 | 偏易 | 标准 | 偏难 |
|------|------|------|------|
| **A1** (入门) | Lexile A1- | Lexile A1 | Lexile A1+ |
| **A2** (基础) | Lexile A2- | Lexile A2 | Lexile A2+ |
| **B1** (中级) | Lexile B1- | Lexile B1 | Lexile B1+ |
| **B2** (中高级) | Lexile B2- | Lexile B2 | Lexile B2+ |
| **C1** (高级) | Lexile C1- | Lexile C1 | Lexile C1+ |
| **C2** (精通) | Lexile C2- | Lexile C2 | Lexile C2+ |

### CEFR ↔ Lexile 分组映射

根据评估出的用户水平，推荐书归入对应分组：

| 推荐难度 | 目标分组 | 说明 |
|----------|----------|------|
| 🟢 巩固阅读 | `Lexile {当前级别}-` 或 `Lexile {当前级别}` | 轻松读，建立信心 |
| 🟡 主力阅读 | `Lexile {当前级别}` 或 `Lexile {当前级别}+` | 有点挑战，核心提升 |
| 🔴 挑战阅读 | `Lexile {上一级别}+` | 少量尝试，突破上限 |

例如：用户评估为 B1 → 🟢 归入 `Lexile B1-`，🟡 归入 `Lexile B1`，🔴 归入 `Lexile B1+`。

### 去重规则

**所有以 `Lexile ` 开头的书架分组视为英文分级书库。** 推荐去重时：
- 从 `/shelf/sync` 的 `archive[]` 中找出所有 `name` 以 `Lexile ` 开头的分组
- 合并这些分组的 `bookIds` 作为**已读/已收藏英文书全集**
- 推荐书中凡是 bookId 在此合集中的，标注「📚 已在书架」并默认跳过
- 同一原著的不同版本（不同 bookId 但同书名）也提示可能重复

已有非 Lexile 命名的英文分组（如"英语""英语已读"）：
- 不影响去重逻辑（去重只看 bookId，不看分组名）
- **提示用户**将这些旧分组中的英文书迁移到 Lexile 分组体系

## 核心原则

1. **每次执行都重新获取实时数据。** 禁止依赖缓存或之前的运行结果，用户的划线和书架可能随时变化。
2. **用数据说话。** 词汇级别判定基于实际划线单词的 CEFR 分布，不凭空猜测。
3. **推荐要有理由。** 每本推荐书都说明为什么适合用户当前水平。
4. **只推荐英文原版书。** 过滤掉中译本和中文书。
5. **分级以书架 Lexile 分组为准。** 微信读书书架上的 Lexile 分组是图书分级的唯一权威数据源。已归入某个 Lexile 分组的书，其分级即为该分组对应的级别，**禁止重新判定或建议调整**。本地不保存任何图书分级信息——每次运行从 `/shelf/sync` 实时读取 Lexile 分组数据作为 ground truth。未归入任何 Lexile 分组的新书，Claude 给出推荐分级建议；一旦用户将其加入对应 Lexile 分组，后续运行直接读书架分组，不再重新判定。
6. **谨慎调整已有分组。** 非必要不调整已收录图书的 Lexile 分组。只有当用户明确要求（如"重新评估"、"这本书太简单了"）或发现明显分级错误（如 B2+ 书误入 B1-）时才提出调整建议，并解释理由等待用户确认。
7. **了解读者再推荐。** 词汇数据只能反映语言水平，不能反映阅读偏好。推荐前先收集读者的年龄、职业、兴趣、阅读目标等画像信息，确保推荐书目在内容和主题上也匹配读者。所有画像问题均可跳过，跳过时使用默认画像（成年通用）。

## 工作流

### Step 0: 确定分析目标

**0a. 若用户指定了书名** → 搜索获取 bookId，跳到 Step 1。

**0b. 若用户未指定书名** → 调 `/user/notebooks` 获取全部有笔记的书，
筛选英文书（`language == "en"` 或书名含 "英文版"/"英文原版"），
按划线数降序排列，展示列表让用户选择：

```
你有以下英文书有划线笔记：
1. 小王子（英文版） — 140 条划线
2. Animal Farm（英文版） — 53 条划线
...
选择要分析的书（输入编号）：
```

若只有一本英文书有划线，直接使用，无需询问。

### Step 1: 获取划线词汇

调 `/book/bookmarklist`（参数 `bookId`）获取该书的全部划线。

从返回的 `updated[].markText` 提取单词列表：

```python
import json, sys, re
data = json.load(sys.stdin)
words = []
for item in data.get('updated', []):
    text = item.get('markText', '').strip()
    # 划线是单词而非句子：提取第一个单词
    word = re.split(r'[\s,.;:!?，。；：！？、""''（）\(\)\[\]{}…—\-—\n\r\t]', text)[0]
    if word and len(word) > 1:
        words.append(word)
```

> 注意：微信读书中英文书的划线多数是**单词级别**（用户点按查词后划线），
> 但也可能有短语或句子划线。优先取 `markText` 的首词；若首词长度 ≤1（标点等），取第二词。

### Step 2: 词汇分析 — 阅读水平评估

> **参考指南**: 详细的分级标准和判定规则见 `references/cefr-guide.md`。

对每个单词进行 CEFR 分级（A2/B1/B2/C1/C2），综合以下信号：

1. **词频信号**：高频日常词 → 低级，低频文学词 → 高级
2. **形态信号**：基础词根 → 低级，复杂派生/复合 → 高级
3. **语域信号**：口语化 → 低级，文学/学术/古雅 → 高级
4. **认知信号**：具体名词 → 低级，抽象概念 → 高级

**输出分析报告**：

```
## 词汇难度分布

| CEFR | 数量 | 占比 | 示例 |
|------|------|------|------|
| A2 | 12 | 8% | amuse, confess, tragedy |
| B1 | 49 | 35% | reputation, eternal, poetic |
| B2 | 56 | 40% | stately, dejection, bewildered |
| C1 | 23 | 17% | ephemeral, balderdash, peevishly |

## 生词类型特征
- 文学描写词: 40% (lugubrious, coquettish, voluminous...)
- 情感/心理词: 30% (dejection, resentfulness, remorse...)
- 高级副词: 15% (indulgently, consumingly, thriftily...)
- 古雅/诗歌用词: 10% (blest, clad, abodes...)
- 基础词汇: 5% (amuse, confess, fair)

## 水平判定: B1（中级）→ B2 过渡期

**依据**: B1+B2 词共占 75%，C1 词 17% 且多为文学性词汇（属正常标记行为）。
划线密度约 1.4%（140条/10000词），说明阅读时需要词典但能理解主线。
```

#### 文风诊断（Step 2 的延伸分析）

词汇分析除了判定 CEFR 水平，还能揭示**你已经适应的文风类型**——这比 Lexile 数字更能预测你会觉得哪本书"好读"。

从划线单词的组成推断源书的文风特征：

| 划线词特征 | 对应文风 | 推荐方向 |
|-----------|----------|----------|
| 多为动作动词 + 具体名词（puff, hoisted, pulley, strap） | **短句动作型**（海明威式） | 优先推荐原生英语短句作家 |
| 多为对话标签 + 情态动词（retorted, obliged, confessed） | **对话驱动型**（阿加莎式） | 优先推荐推理/悬疑/对话密集作品 |
| 多为抽象名词 + 心理动词（dejection, reverie, pondered, meditate） | **哲思内省型**（黑塞/卡夫卡式） | 这类书对用户可能偏难 |
| 多为自然景物词（dune, sprout, vineyards, ridges） | **描写密集型**（杰克·伦敦式） | 描写占比高，需评估是否适合 |
| 多为古雅/诗歌用词（blest, clad, abodes, yonder） | **古雅翻译型**（19 世纪经典英译本） | 维多利亚时代英文，难度偏高 |

**关键规则**：
- 划线 = 用户不认识的词 = **这本书的文风弱点**。大量古雅词被划线 → 古雅文风是用户的障碍，不是舒适区。
- 划线少的书 = 用户读得顺 → 分析该书的文风类型，这是用户的**舒适文风**。
- **用户主观反馈优先于 Lexile**：用户说"A 比 B 容易"为真，用这个反馈反推舒适文风，调整后续推荐。
- **原生英语 vs 翻译**：原生英语（海明威、斯坦贝克）通常比翻译英语（加缪英译、卡夫卡英译）句子更自然。但现代译本（如加缪的 Stuart Gilbert 译本）可以与原生英语一样流畅。

**输出格式**（紧接 Step 2 报告之后）：

```
## 文风诊断
- 划线词类型：XX% 描写词、XX% 对话词、XX% 抽象词
- 推断源书文风：短句动作型 / 对话驱动型 / 哲思内省型 / 描写密集型
- 舒适区推测：XXX 型文风最匹配
- 应避开：XXX 型（大量此类词被标记）
```

### Step 2.5: 读者画像快速调查（可选）

词汇数据能告诉你「能读什么」，但不能告诉你「想读什么」。
在搜索推荐书目之前，收集读者的背景信息以精准匹配。

**一次性提问**（用户可逐条回答、部分回答或直接跳过）：

```
在推荐之前，先了解一下你的偏好（可跳过任意问题）：

1. 年龄段？ [青少年(<18) / 青年(18-25) / 中青年(26-40) / 中年(41-60) / 60+]
2. 职业/行业？ [IT/科技 / 金融 / 医疗 / 教育 / 创意 / 工程 / 其他]
3. 喜欢的类型？ [文学经典 / 悬疑推理 / 科幻 / 哲学历史 / 商业经管 / 不限] ← 可多选
4. 阅读目标？ [提升英语 / 文学鉴赏 / 职业发展 / 轻松消遣 / 综合]
5. 偏好的篇幅？ [短篇集 / 中篇(~5万字) / 长篇(~10万字) / 巨著 / 不限]

你可以直接回复"都行"或"跳过"使用默认推荐。
```

**默认画像**（用户跳过时使用）：
- 年龄段：中青年（26-40），成年通用推荐
- 职业：不限，不添加职业关键词
- 类型：不限，均衡覆盖各类
- 目标：综合
- 篇幅：偏好中长篇

**画像如何影响后续步骤**：

| 维度 | 影响 Step 3b（书单搜索） | 影响 Step 3c（泛搜索） | 影响 Step 3e（过滤） |
|------|--------------------------|------------------------|----------------------|
| 年龄段 ≥ 41 | 排除纯儿童向（Charlotte's Web、安徒生童话、绿野仙踪、柳林风声），保留成人也读的文学经典（麦田里的守望者、杀死一只知更鸟） | — | 排除儿童文学（动物主角/童话语气），保留青少年主角但成人主题的文学经典 |
| 年龄段 ≤ 17 | — | — | 排除成人黑暗主题 |
| IT/科技 | 优先反乌托邦/科幻 | 加搜 `英文原版 科幻` | — |
| 金融 | 优先商业叙事类 | 加搜 `英文原版 商业` | — |
| 医疗 | 优先医学人文类 | 加搜 `英文原版 医学` | — |
| 悬疑推理 | 阿加莎/钱德勒置顶 | 加搜 `英文原版 悬疑推理` | — |
| 科幻 | 奥威尔/赫胥黎/布拉德伯里置顶 | 加搜 `英文原版 科幻` | — |
| 哲学历史 | 黑塞/加缪/历史叙事置顶 | 加搜 `英文原版 哲学` `英文原版 历史` | — |
| 商业经管 | 商业叙事类置顶 | 加搜 `英文原版 商业` | — |
| 提升英语 | — | `中英 双语` 搜索 count 加倍 | 优先双语版 |
| 短篇集 | — | 加搜 `英文 短篇` | 优先短篇/中篇 |
| 巨著 | — | — | 优先长篇 |

**推荐理由措辞**：Step 4d 输出时，根据画像中的「阅读目标」调整每本书的推荐语风格：

| 阅读目标 | 推荐语侧重 |
|----------|-----------|
| 提升英语 | 强调句式复杂度、词汇复现率、是否双语 |
| 文学鉴赏 | 强调文学价值、语言风格、作者地位 |
| 职业发展 | 强调行业相关性、实用表达、思维启发 |
| 轻松消遣 | 强调节奏感、可读性、故事吸引力 |
| 综合 | 均衡覆盖以上维度 |

此外，如用户填写了职业，推荐语中可提及（如"作为 IT 从业者，1984 对监控与技术的反思会让你感同身受"）。

**边界情况**：
- 用户直接跳过 → 使用默认画像，不追问
- 用户只回答部分问题 → 已答项采用用户答案，未答项使用默认值
- 用户多选类型（如"悬疑推理+科幻"）→ 合并两个类型的搜索关键词和置顶规则，推荐时均衡覆盖
- 用户自由文本回答 → Claude 解析到最接近的标准选项（如"做运维的"→IT/科技，"想看烧脑的"→悬疑推理+科幻）
- 同一会话重复触发 → 可以复用之前的画像回答（记忆在对话上下文中），提示"是否使用上次的偏好？"
- 用户中途说"其实我是做XX的" → 更新对应维度，增量调整推荐（不一定全量重搜）
- 用户说"重新回答偏好问题" → 重跑 Step 2.5

### Step 3: 搜索推荐书目

采用**分层搜索策略**，按英文命中率从高到低排列。关键发现：以英文书为种子调 `/book/similar` 英文命中率 **85%**，
远超 `/book/recommend`（基于全库阅读历史，英文仅 15%）。

#### 3a. 主力：以英文书为种子的相似推荐（英文命中率 ~85%）⭐

**这是最高效的英文书发现方式。** 对用户**每本有划线的英文书**各调一次 `/book/similar`，
合并去重后得到大量同语言好书。

```bash
# 对每本英文书并行调用（必须带 maxIdx=0，否则报参数格式错误）
curl ... -d '{"api_name":"/book/similar","bookId":"<bookId>","count":30,"maxIdx":0,...}'
```

从回包 `booksimilar.books[].book.bookInfo` 提取 `bookId, title, author, language`。

**为什么有效**：推荐算法以书籍向量做相似匹配，英文种子 → 英文结果。用户读过 N 本英文书
就有 N 个高精度推荐源。

**翻页**：若结果 ≥30 本，用最后一条的 `idx` 作为 `maxIdx`，带上 `sessionId` 翻页。
通常翻 1 页即可（每页 30 本），英文书密度已经足够。

**去重合并后**，按难度档位分组进入候选池。

#### 3b. 补充：已知适读书单搜索

similar 覆盖不到的冷门/特定版本，用关键词精确搜索补全。

> ⚠️ **WeRead 中文书名搜索不可靠**：中文关键词如"鲁宾逊漂流记 英文版""月亮与六便士 英文版"
> 在 `/store/search` 中经常返回空结果（书名格式不统一："XXX（英文版）" vs "XXX（英文原版）" vs "XXX 英文朗读版"）。
> **优先用作者名 + 英文书名搜索**（如 `Steinbeck`、`Siddhartha`）。
> 中文书名搜索仅作为补充尝试，不应阻塞推荐流程——即使全空也不影响结果（similar + 泛搜索已覆盖大部分）。

> ⚠️ **以下列表仅用于关键词搜索发现书籍，不代表最终分级。**
> 每本书的 Lexile 级别在 Step 3e 中根据书架已有 Lexile 分组数据（已归入的）或 Claude 分级判断（新书）确定，
> 可能与列表所在的难度分组不同。

根据用户水平，搜索以下书名（按难度分组）：

#### 3e. 去重与过滤

合并所有来源的搜索结果：
- 按 `bookId` 去重
- **过滤非英文书**：中译本、法语等其他语言
- **Lexile 分组去重**：从 `/shelf/sync` 的 `archive[]` 中找出所有以 `Lexile ` 开头的分组，合并其 `bookIds` 作为已收藏英文书全集。推荐书在此合集中的 → 标注「📚 已在 Lexile 书架」并跳过
- 同时检查 `books[]` 中是否有同名不同版本的书 → 提示"可能已有其他版本"
- **去重同书不同版本**：同一原著的不同版本/语言版本只保留一本最合适的
- **确定每本书的 Lexile 分级**：对每本候选书，先查书架 Lexile 分组中是否已有该书。已在某分组 → 以该分组的级别为准，**不再重新判定**。未在任何 Lexile 分组中 → Claude 综合以下因素给出推荐分组：① WebFetch/WebSearch 查官方 Lexile 分数（hub.lexile.com）② 同作者其他书的已入库分组 ③ 时代英语特征（维多利亚→+1档，现代→-1档）④ 同类已入库书的参照对比。分级判定后在同一会话内不得更改；**绝对禁止**凭印象反复调整
- **不推荐过难的书**：19 世纪长篇经典（狄更斯、简·奥斯汀原著、艾米莉·勃朗特、莎士比亚）标注为「挑战」或排除，归入比用户当前级别高两档以上的分组
- **优先保留**：双语版 > 短篇集 > 现代小说 > 经典文学
- **年龄过滤**：年龄段 ≥ 41 → 排除纯儿童文学（动物主角童话、迪士尼配套书、YA 校园言情），但**保留**青少年主角但成人广泛阅读的文学经典（如 The Catcher in the Rye、To Kill a Mockingbird）；年龄段 ≥ 26 → 排除低幼向/童话向；年龄段 ≤ 17 → 排除含成人黑暗主题的书
- **篇幅偏好**：用户指定短篇/中篇 → 同难度下优先选 3-8 万词作品；用户指定巨著 → 优先展示 15 万词以上长篇
- **内容定向**：根据画像中的类型偏好，同难度下优先展示匹配类型的书；用户多选类型时均衡覆盖
- **similar 来源加权**：来自 similar 推荐的书（同一向量空间）置信度更高，优先展示
- **作者-分组亲和（书架隐式反馈）**：从 Lexile 书架中统计已有书的「作者→分组」分布。若某作者已有 ≥3 本归入同一分组，该作者的新书**轻微倾向**同一分组。**这是弱信号，不覆盖单书特征**：篇幅、年代、题材、文风差异仍可改变最终分级。亲和规则仅在同级别的边界情况（如B1还是B1-）时作为 tiebreaker
- **书架文风偏好**：统计 Lexile 书架中高频出现的作者文风类型（海明威→短句型、阿加莎→对话型），搜索时额外搜索同类文风作者名（见 Step 3c 作者名搜索表），推荐时同类文风的书优先展示

### Step 4: 分级推荐与书架分组

> **分级来源**：每本书的 Lexile 级别以书架 Lexile 分组为准（已在分组中的书）或 Claude 新书判定（未归入任何分组的新书）。

## 边界情况处理

| 情况 | 处理 |
|------|------|
| 没有英文书划线 | 提示用户先在微信读书中阅读英文书并标记生词 |
| 划线数 < 20 | 样本太少，降低判定置信度，给出参考范围并建议多标记后再评估 |
| 划线多是短语/句子（非单词） | 尝试用 NLP 提取关键词，或提示用户这个 skill 更适合单词级划线 |
| 搜索英文书结果很少 | 扩大搜索范围（scope=0 全部），尝试英文关键词 + 中文关键词组合 |
| 推荐书已在某个 Lexile 分组中 | 标注 `📚 Lexile XX` 并跳过，不重复推荐 |
| 推荐书已在书架但不在 Lexile 分组 | 提示"已在书架但未归入 Lexile 分组"，建议迁移 |
| 用户书架无 Lexile 分组 | 提示创建首批 Lexile 分组（最少 3 个：当前级别 -, 标准, +），等用户确认后再继续 |
| 用户有旧英文分组（"英语""英语已读"等） | 提示迁移到 Lexile 体系，但不阻塞推荐流程 |
| 用户跳过画像调查 | 使用默认画像（中青年、不限类型、综合目标、中长篇），不追问 |
| 用户只回答部分画像问题 | 已答项采用用户答案，未答项使用默认值 |
| 用户用自由文本描述偏好 | Claude 解析到最接近的标准选项（如"做运维的"→IT/科技，"想看烧脑的"→悬疑推理+科幻） |
| 用户中途补充画像信息 | 更新对应维度，增量调整推荐（不一定全量重搜） |
| 用户画像与词汇水平矛盾 | 以词汇水平为准定难度，画像只影响类型/主题选择 |
| 返回用户（同一会话） | 复用已收集的画像，提示"是否使用上次的偏好？"；可说"更新偏好"修改 |
| 返回用户（新会话） | 重新提问（5 个问题开销很低） |
