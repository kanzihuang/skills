# Book Recommend — 英文阅读水平评估与图书推荐

根据微信读书英文书划线生词，自动评估英文阅读水平（CEFR），搜索并推荐匹配难度的英文原版书，
提供深度链接一键跳转添加到书架。

## 前置条件

- `weread-skills` 已安装
- `WEREAD_API_KEY` 环境变量已设置
- 微信读书中至少有一本英文书有划线记录

## 书架分组命名规范

微信读书书架中的英文书按 **Lexile + CEFR** 规则命名分组，实现难度分级管理。

### 命名格式

```
Lexile {CEFR级别}{难度偏移}
```

| 难度偏移 | 含义 | 示例 |
|----------|------|------|
| `-` | 该级别中偏易 | `Lexile B1-` |
| (无后缀) | 该级别标准难度 | `Lexile B1` |
| `+` | 该级别中偏难/过渡到下一级 | `Lexile B1+` |

### 完整分组列表（18 个）

| 级别 | 偏易 | 标准 | 偏难 |
|------|------|------|------|
| **A1** (入门) | Lexile A1- | Lexile A1 | Lexile A1+ |
| **A2** (基础) | Lexile A2- | Lexile A2 | Lexile A2+ |
| **B1** (中级) | Lexile B1- | Lexile B1 | Lexile B1+ |
| **B2** (中高级) | Lexile B2- | Lexile B2 | Lexile B2+ |
| **C1** (高级) | Lexile C1- | Lexile C1 | Lexile C1+ |
| **C2** (精通) | Lexile C2- | Lexile C2 | Lexile C2+ |

### CEFR ↔ Lexile 分组映射

根据评估出的用户水平，推荐书归入对应分组：

| 推荐难度 | 目标分组 | 说明 |
|----------|----------|------|
| 🟢 巩固阅读 | `Lexile {当前级别}-` 或 `Lexile {当前级别}` | 轻松读，建立信心 |
| 🟡 主力阅读 | `Lexile {当前级别}` 或 `Lexile {当前级别}+` | 有点挑战，核心提升 |
| 🔴 挑战阅读 | `Lexile {上一级别}+` | 少量尝试，突破上限 |

例如：用户评估为 B1 → 🟢 归入 `Lexile B1-`，🟡 归入 `Lexile B1`，🔴 归入 `Lexile B1+`。

### 去重规则

**所有以 `Lexile ` 开头的书架分组视为英文分级书库。** 推荐去重时：
- 从 `/shelf/sync` 的 `archive[]` 中找出所有 `name` 以 `Lexile ` 开头的分组
- 合并这些分组的 `bookIds` 作为**已读/已收藏英文书全集**
- 推荐书中凡是 bookId 在此合集中的，标注「📚 已在书架」并默认跳过
- 同一原著的不同版本（不同 bookId 但同书名）也提示可能重复

已有非 Lexile 命名的英文分组（如"英语""英语已读"）：
- 不影响去重逻辑（去重只看 bookId，不看分组名）
- **提示用户**将这些旧分组中的英文书迁移到 Lexile 分组体系

## 核心原则

1. **每次执行都重新获取实时数据。** 禁止依赖缓存或之前的运行结果，用户的划线和书架可能随时变化。
2. **用数据说话。** 词汇级别判定基于实际划线单词的 CEFR 分布，不凭空猜测。
3. **推荐要有理由。** 每本推荐书都说明为什么适合用户当前水平。
4. **只推荐英文原版书。** 过滤掉中译本和中文书。
5. **分级稳定优先。** 每本书的 Lexile 分级必须从 `references/book-levels.md` 查找；若该书记录在案，直接沿用已有分级，不得在同一会话或后续会话中无新证据更改。若未收录，按该文档规定的流程判定并追加到文档中。**绝对禁止**凭印象临时给书定级——临时定级是分级漂移的根源。

## 工作流

### Step 0: 确定分析目标

**0a. 若用户指定了书名** → 搜索获取 bookId，跳到 Step 1。

**0b. 若用户未指定书名** → 调 `/user/notebooks` 获取全部有笔记的书，
筛选英文书（`language == "en"` 或书名含 "英文版"/"英文原版"），
按划线数降序排列，展示列表让用户选择：

```
你有以下英文书有划线笔记：
1. 小王子（英文版） — 140 条划线
2. Animal Farm（英文版） — 53 条划线
...
选择要分析的书（输入编号）：
```

若只有一本英文书有划线，直接使用，无需询问。

### Step 1: 获取划线词汇

调 `/book/bookmarklist`（参数 `bookId`）获取该书的全部划线。

从返回的 `updated[].markText` 提取单词列表：

```python
import json, sys, re
data = json.load(sys.stdin)
words = []
for item in data.get('updated', []):
    text = item.get('markText', '').strip()
    # 划线是单词而非句子：提取第一个单词
    word = re.split(r'[\s,.;:!?，。；：！？、""''（）\(\)\[\]{}…—\-—\n\r\t]', text)[0]
    if word and len(word) > 1:
        words.append(word)
```

> 注意：微信读书中英文书的划线多数是**单词级别**（用户点按查词后划线），
> 但也可能有短语或句子划线。优先取 `markText` 的首词；若首词长度 ≤1（标点等），取第二词。

### Step 2: 词汇分析 — 阅读水平评估

> **参考指南**: 详细的分级标准和判定规则见 `references/cefr-guide.md`。

对每个单词进行 CEFR 分级（A2/B1/B2/C1/C2），综合以下信号：

1. **词频信号**：高频日常词 → 低级，低频文学词 → 高级
2. **形态信号**：基础词根 → 低级，复杂派生/复合 → 高级
3. **语域信号**：口语化 → 低级，文学/学术/古雅 → 高级
4. **认知信号**：具体名词 → 低级，抽象概念 → 高级

**输出分析报告**：

```
## 词汇难度分布

| CEFR | 数量 | 占比 | 示例 |
|------|------|------|------|
| A2 | 12 | 8% | amuse, confess, tragedy |
| B1 | 49 | 35% | reputation, eternal, poetic |
| B2 | 56 | 40% | stately, dejection, bewildered |
| C1 | 23 | 17% | ephemeral, balderdash, peevishly |

## 生词类型特征
- 文学描写词: 40% (lugubrious, coquettish, voluminous...)
- 情感/心理词: 30% (dejection, resentfulness, remorse...)
- 高级副词: 15% (indulgently, consumingly, thriftily...)
- 古雅/诗歌用词: 10% (blest, clad, abodes...)
- 基础词汇: 5% (amuse, confess, fair)

## 水平判定: B1（中级）→ B2 过渡期

**依据**: B1+B2 词共占 75%，C1 词 17% 且多为文学性词汇（属正常标记行为）。
划线密度约 1.4%（140条/10000词），说明阅读时需要词典但能理解主线。
```

### Step 3: 搜索推荐书目

采用**分层搜索策略**，按英文命中率从高到低排列。关键发现：以英文书为种子调 `/book/similar` 英文命中率 **85%**，
远超 `/book/recommend`（基于全库阅读历史，英文仅 15%）。

#### 3a. 主力：以英文书为种子的相似推荐（英文命中率 ~85%）⭐

**这是最高效的英文书发现方式。** 对用户**每本有划线的英文书**各调一次 `/book/similar`，
合并去重后得到大量同语言好书。

```bash
# 对每本英文书并行调用（必须带 maxIdx=0，否则报参数格式错误）
curl ... -d '{"api_name":"/book/similar","bookId":"<bookId>","count":30,"maxIdx":0,...}'
```

从回包 `booksimilar.books[].book.bookInfo` 提取 `bookId, title, author, language`。

**为什么有效**：推荐算法以书籍向量做相似匹配，英文种子 → 英文结果。用户读过 N 本英文书
就有 N 个高精度推荐源。

**翻页**：若结果 ≥30 本，用最后一条的 `idx` 作为 `maxIdx`，带上 `sessionId` 翻页。
通常翻 1 页即可（每页 30 本），英文书密度已经足够。

**去重合并后**，按难度档位分组进入候选池。

#### 3e. 去重与过滤

合并所有来源的搜索结果：
- 按 `bookId` 去重
- **过滤非英文书**：中译本、法语等其他语言
- **Lexile 分组去重**：从 `/shelf/sync` 的 `archive[]` 中找出所有以 `Lexile ` 开头的分组，合并其 `bookIds` 作为已收藏英文书全集。推荐书在此合集中的 → 标注「📚 已在 Lexile 书架」并跳过
- 同时检查 `books[]` 中是否有同名不同版本的书 → 提示"可能已有其他版本"
- **去重同书不同版本**：同一原著的不同版本/语言版本只保留一本最合适的
- **查 `references/book-levels.md` 确定每本书的 Lexile 分级**：对每本候选书，先查文档中是否已有记录。有记录 → 直接使用已有分级；无记录 → 按文档规定的流程（Lexile 分数 + ESL 修正因子）判定，然后用 Edit 追加到文档中。**绝对禁止**凭印象临时定级——临时定级是分级漂移的根源
- **不推荐过难的书**：19 世纪长篇经典（狄更斯、简·奥斯汀原著、艾米莉·勃朗特、莎士比亚）标注为「挑战」或排除，归入比用户当前级别高两档以上的分组
- **优先保留**：双语版 > 短篇集 > 现代小说 > 经典文学
- **similar 来源加权**：来自 similar 推荐的书（同一向量空间）置信度更高，优先展示

### Step 4: 分级推荐与书架分组

> **分级映射规则**：每本书的绝对 Lexile 级别从 `references/book-levels.md` 获取。
> 然后将绝对级别映射到推荐档位（🟢巩固 / 🟡主力 / 🔴挑战），映射规则见 `references/book-levels.md` 末尾的「推荐时的级别映射」表。
> 输出时，每本书标注其绝对 Lexile 分组（如 `Lexile B1`），档位颜色由映射规则决定。

#### 4a. 获取书架分组信息与 Lexile 分组就绪检查

调 `/shelf/sync` 获取当前书架数据：
- `books[]`：已有电子书列表（用于去重）
- `archive[]`：现有分组/书单（`name` + `bookIds`）

**Lexile 分组就绪检查**：

1. 从 `archive[]` 中找出所有以 `Lexile ` 开头的分组
2. 若已有 Lexile 分组 → 展示现有分组，用户可选择已有分组或新建
3. 若**完全没有 Lexile 分组** → 根据用户水平推荐需要创建的分组：

```
你的书架还没有 Lexile 分级分组。建议创建以下分组来管理英文书：

根据你的水平 (B1)，建议优先创建：
  ▸ Lexile B1-  (偏易，巩固阅读)
  ▸ Lexile B1   (标准，主力阅读)
  ▸ Lexile B1+  (偏难，挑战阅读)

后续扩展：
  ▸ Lexile A2+  (已掌握级别，回溯经典)
  ▸ Lexile B2-  (下一阶段过渡)

请在微信读书 App 中手动创建这些分组（书架 → 分组管理 → 新建分组），
完成后回复"已创建"，我将继续推荐。
```

用户确认分组就绪后继续。

**Lexile 已就绪时**，根据 Step 2 评估出的用户水平，自动匹配目标分组：

```
你的 Lexile 书架：
  Lexile B1- (12本)  Lexile B1 (8本)  Lexile B1+ (3本)

本次推荐将归入：
  🟢 巩固 → Lexile B1-
  🟡 进阶 → Lexile B1
  🔴 挑战 → Lexile B1+

确认使用这些分组？(输入 y 确认，或输入分组名修改)
```

#### 4b. 将推荐书加入书架分组

**微信读书 API 目前不提供添加书籍到书架或分组的写操作接口**，
因此采用**深度链接方案**：

1. 每本推荐书生成 `weread://reading?bId={bookId}` 深度链接
2. 用户点击链接 → 微信读书 App 打开该书
3. 在 App 中点击「加入书架」→ 拖入目标分组

输出时列出所有推荐书，每本附带 `[📖 打开加入](weread://...)` 链接。
若用户指定了分组，输出顶部注明目标分组名称：

```
> 📂 目标分组：英语学习
> 点击每本书下方的链接 → 在微信读书 App 中打开 → 加入书架 → 拖入「英语学习」分组
```

#### 4c. 辅助：检查推荐书是否已在书架

完成推荐后，可再次调 `/shelf/sync` 检查哪些推荐书已被用户加入书架，
标注 ✅ 已加入 / ⏳ 待加入 状态。这个检查只在用户主动要求时做，
不要每次都自动做（避免频繁调接口）。

#### 4d. 输出格式

三级推荐 + Lexile 分组 + 深度链接：

```
## 📚 为你推荐的英文原版书

> 📂 目标分组: Lexile B1- / Lexile B1 / Lexile B1+
> 点击书籍链接 → 微信读书 App 打开 → 加入书架 → 拖入对应 Lexile 分组

### 🟢 巩固区 → Lexile B1-
| # | 书名 | 作者 | 评分 | 打开 |
|---|------|------|------|------|
| 1 | 动物庄园 (英文版) | 乔治·奥威尔 | 88.3 | [📖](weread://reading?bId=830009) |
| 2 | 老人与海 (英文原版) | 海明威 | 91.3 | [📖](weread://reading?bId=3300144556) |
...

> 为什么推荐：句子简洁，词汇量适中。Animal Farm 用词精准但不过度文学化，是 B1→B2 的经典跳板。

### 🟡 进阶区 → Lexile B1
...

### 🔴 挑战区 → Lexile B1+
...
```

推荐书如已在某个 Lexile 分组中，标注 `📚 Lexile B1` 而非重复推荐。

## 📖 阅读策略
1. 70% 时间读 🟢🟡，30% 尝试 🔴
2. 优先选中英双语版或短篇集
3. 继续用划线功能精读
4. 读完一批后重新评估，水平提升后迁移到下一级 Lexile 分组
```

> 推荐的每本书都标注目标 **Lexile 分组**，附 **weread:// 深度链接**。
> 用户点击后在微信读书 App 打开 → 加入书架 → 拖入对应的 Lexile 分组。

### Step 5（可选）: 持续追踪

用户下次使用此技能时，自动对比：
- 哪些推荐书已加入书架（从 `/shelf/sync` 获取）
- 新划线词汇的难度变化
- 推荐是否仍匹配当前水平

### 书架迁移（现有英文书归类）

当用户书架上有英文书但未归入 Lexile 分组时（如在「英语」「英语已读」等旧分组中），
生成迁移建议表。**每本书的 Lexile 级别严格从 `references/book-levels.md` 查询，不得临时判定**：

```
| 现有英文书 | → 归入 |
|-----------|--------|
| Charlotte's Web | Lexile B1- |
| 小王子 | Lexile B1 |
| To Kill a Mockingbird | Lexile B2 |
```

对于 `book-levels.md` 中未收录的书，按新增书籍流程判定后追加到文档，再生成迁移建议。

## 边界情况处理

| 情况 | 处理 |
|------|------|
| 没有英文书划线 | 提示用户先在微信读书中阅读英文书并标记生词 |
| 划线数 < 20 | 样本太少，降低判定置信度，给出参考范围并建议多标记后再评估 |
| 划线多是短语/句子（非单词） | 尝试用 NLP 提取关键词，或提示用户这个 skill 更适合单词级划线 |
| 搜索英文书结果很少 | 扩大搜索范围（scope=0 全部），尝试英文关键词 + 中文关键词组合 |
| 推荐书已在某个 Lexile 分组中 | 标注 `📚 Lexile XX` 并跳过，不重复推荐 |
| 推荐书已在书架但不在 Lexile 分组 | 提示"已在书架但未归入 Lexile 分组"，建议迁移 |
| 用户书架无 Lexile 分组 | 提示创建首批 Lexile 分组（最少 3 个：当前级别 -, 标准, +），等用户确认后再继续 |
| 用户有旧英文分组（"英语""英语已读"等） | 提示迁移到 Lexile 体系，但不阻塞推荐流程 |

## 接口调用规范

遵循 weread-skills 的接口调用规范：
- 统一入口 `POST https://i.weread.qq.com/api/agent/gateway`
- Header `Authorization: Bearer $WEREAD_API_KEY`
- 业务参数平铺在 body 顶层，必须带 `skill_version: "1.0.3"`
- 每次调用接口前阅读对应说明文件（search.md / notes.md / shelf.md / discover.md）

### 接口注意事项

| 接口 | 注意 |
|------|------|
| `/book/similar` | **必须传 `maxIdx`**（初始为 `0`），否则报 `-2003 参数格式错误`。翻页时用最后一条的 `idx` 作为新 `maxIdx`，同时传回包中的 `sessionId` |
| `/book/recommend` | 基于用户全库阅读历史（非仅英文书），英文命中率约 15%。用 `count=50` 批量拉取，翻页用 `searchIdx` 作为 `maxIdx` |
| `/store/search` | 英文书名搜索可能无结果，需要同时用**中文译名**搜（如"怦然心动"而非"Flipped"）。已知可搜到的英文书多数归类在"教育学习-外语"下 |
